# Contributing to the SDM Framework

**ElosiaEcosystem Inc. | Structured Distance Measurement**

---

## How This Repository Works

The SDM framework is an **open research corpus** — not a software project seeking pull requests for code contributions. The primary mode of engagement is intellectual: challenging concepts, identifying gaps, surfacing limitations, and extending the framework's ideas into new domains.

This document explains how to participate constructively.

---

## Ways to Engage

### 1. Challenge a Concept — GitHub Discussions

The framework is built around adversarial pressure. If you think a concept is wrong, incomplete, or trapped in an assumption it failed to escape — say so.

**Open a Discussion to:**
- Challenge the Consensus Trap theory
- Question whether D3's empirical anchoring is too conservative
- Propose a different framing for the Promotion Boundary
- Identify a domain where the framework's assumptions break down
- Surface a limitation not already documented

**Discussion rules:**
- No source, no claim — bring evidence or reasoning, not assertion
- Critique the framework, not the person
- Engage with the concept at the paradigm level, not the implementation level

[→ Open a Discussion](https://github.com/GP1008/SDM/discussions)

---

### 2. Report an Inconsistency — GitHub Issues

If you find a contradiction between documents, a logical gap in the framework, or a claim that does not hold under scrutiny — open an issue.

**Open an Issue to:**
- Flag a logical inconsistency in the framework
- Identify a claim that lacks sufficient grounding
- Point out a gap between the manifesto and the exchange transcripts
- Report a broken link or missing document

**Issue format:**
```
Title: [Document] — Brief description of the inconsistency

Document: which document contains the issue
Section: which section
Claim: what the document states
Problem: why it is inconsistent or unsupported
Suggestion: (optional) how it might be corrected
```

[→ Open an Issue](https://github.com/GP1008/SDM/issues)

---

### 3. Reference and Cite — Academic Use

The conceptual framework is released under CC BY-NC 4.0. Academic researchers are encouraged to:

- Cite the framework in papers
- Reference the Disruptor Exchange model in AI governance research
- Build on the Consensus Trap theory in epistemology or science studies
- Apply SDM concepts to other research domains

**Citation format:**
```
Phillips, G. (2026). Structured Distance Measurement: A Multi-Agent Framework
for Mapping the Edges of What Is Considered Correct.
ElosiaEcosystem Inc. https://github.com/GP1008/SDM
```

---

### 5. Share Exchange Transcripts

If you run your own Disruptor Exchange — using the role definitions and five-step protocol described in the manifesto — the framework benefits from your transcripts.

**To share exchange transcripts:**
1. Format them following the pattern in [`/exchanges`](exchanges/README.md)
2. Open a Discussion linking to your transcript
3. Include: research domain, model assignments, D3 SDM output, and any calibration findings

External exchange transcripts will not be merged into the main repository but will be referenced in Discussions for the benefit of the research community.

## What This Repository Is Not

- Not a software project — there is no code to contribute
- Not a place to submit general AI opinions or proposals
- Not a platform for marketing competing frameworks
- Not accepting contributions to the proprietary operational substrate

The operational governance infrastructure — orchestration runtime, BDRM enforcement, MicroManager/TAG systems — is proprietary and not open for contribution. See [LICENSING.md](LICENSING.md) for the complete boundary.

---

## Conduct

This is a research forum, not a debate competition. Contributions that demonstrate the framework is wrong are as valuable as contributions that confirm it is right. The system is built on adversarial pressure — that extends to how the research community engages with it.

Expected behavior:
- Source your claims
- Engage with the strongest version of the opposing argument
- Preserve dissenting views rather than dismissing them
- Acknowledge when a point lands
